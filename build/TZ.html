Общие требования

  1. Верстка должна быть адаптивной. Переход в мобильный режим на точке 768px. Использовать сетку Bootstrap.
  2. Поддержка браузеров: IE10+ .Ghrome ,Opera ,Mozilla ,Safari - 2 последние версии
  3. Проверка верстки PerfectPixel'ем.
  4. Все элементы ,с которыми предполгается взаимодействие ,имеют hover-эффекты. Некоторые hover-эффекты показаны в PSD скрытыми слоями ,выделенными желтым цветом. Смотреть слови внимательно.
  5. Картинку оптимизировать - сжать с помощью Gulp
  6. Шрифты подключать стандартным способом - через @font-face
  7. При верстке использовать Github и пушить все изменения в ветку gh-pages
  8. В целом все должно получиться как на PSD макете. Никаких отклоений от макета быть не должно

HTML и CSS

  1. Логичные имена классов. Не использовать транслиты
  2. Классы используем для стилизации ,айдишники для JS
  3. Указывать альтернативные шрифты и тип семейства
  4. Использовать normalize.css
  5. Не использовать инлайн-стили
  6. Использовать псевдоэлементы before и after для декоративных элементов
  7. HTML файл должен быть чистым и аккуратным. Соблюдать иерархию отступов.

Препроцессор SASS

  1. Правильная архитектура файлов. Не кидать все стили в один файл ,а разбивать на логические части. Например - main ,fonts, vars, mixins, media и т.д.
  2. Соблюдать все отступы ,чтобы в документе не было никаких дыр и все выглядело ровно.
  3. Компилировать SASS в минифициорванный файлик style.min.css
  4. Использовать любые liveReload. Желательно плагин для Gulp ,который я использую при верстке
  5. Bootstrap подключать не через CDN. Качаем исходники на SASS с оф. сайта или гитхаба. Заходим в файлик всех подключений и убираем все импорты кроме сетки (grid) ,переменных и миксинов. Подробная информация есть в продвинутом интенсиве HTML Academy в лекции про Фреймворки

Детали

  1. Все цвета брать не пипеткой ,а через палитру слоя
  2. Оболочка IPhone и картинка внутри него - 2 разных изображения! На точке 768px оболочка IPhone должна пропасть ,а картинка растянуться на всю ширину
  3. Массонские херни добавлять через background-image. Использовать background-position для их позиционирования
  4. Ромбик перед заголовком преборазовать в SVG и вставить SVG кодом в разметку
  5. У инпутов и кнопок не должно быть аутлайнов в фокусе или при нажатии
  6. Зубчики использовать как паттерн ,повторяющийся по вертикали и прижатый к сетке. При 768px зубчики должны пропадать вместе с боковыми стенками
